# Pika 视频生成平台指南

## 平台概述

Pika 是流行的 AI 视频生成平台，提供:
- **Pika 1.0** - 最新的视频生成模型
- **Pika Labs** - Discord 和 Web 平台
- **Text to Video** - 文生视频
- **Image to Video** - 图生视频
- **Video to Video** - 视频风格转换
- **Video Extend** - 视频扩展

## 关键特性

| 特性 | 描述 |
|------|------|
| 最大时长 | 10秒（可扩展） |
| 分辨率 | 1080p |
| 宽高比 | 16:9, 9:16, 1:1 |
| 图生视频 | ✅ 支持 |
| 视频到视频 | ✅ 支持 |
| 视频扩展 | ✅ 支持 |
| 一致角色 | ✅ 支持 |

## 提示词指南

### 结构建议

```
[主体/角色], [动作/行为], [环境/背景], [时间/天气], [风格/氛围], [镜头/相机], [技术参数]
```

### 示例

**基础示例:**
```
一只熊猫在竹林中吃竹子，阳光透过竹叶，自然清新，写实风格，4K，高质量
```

**高级示例:**
```
未来城市的飞行汽车在高楼间穿梭，霓虹灯光闪烁，赛博朋克风格，电影质感，航拍视角，4K，高质量
```

## 关键参数

| 参数 | 类型 | 范围/选项 | 说明 |
|------|------|-----------|------|
| `duration` | number | 3-10 | 视频时长（秒） |
| `aspect_ratio` | string | "16:9", "9:16", "1:1" | 视频宽高比 |
| `model` | string | "pika-1.0", "pika-0.9" | 使用的模型 |
| `motion` | number | 0-10 | 运动强度 |
| `seed` | number | 任意整数 | 随机种子 |
| `negative_prompt` | string | 文本 | 负面提示词 |

## 最佳实践

### ✅ 推荐做法

1. **简洁有力的描述**
   - 突出关键元素
   - 避免冗余信息
   - 使用精确的词汇

2. **角色一致性**
   - 保持角色描述一致
   - 使用相同的角色特征
   - 可以使用参考图保持一致

3. **风格和氛围**
   - 明确说明风格
   - 描述整体氛围
   - 使用相关的关键词

### ❌ 避免的做法

1. **过长的提示词**
   - 避免提示词过长
   - 避免信息过载

2. **复杂的动作**
   - 避免过于复杂的动作序列
   - 避免不可能的物理动作

3. **负面提示词过多**
   - 避免负面提示词过多
   - 优先使用正面描述

## 完整示例

### 文生视频示例

**输入提示词:**
```
一位魔术师在舞台上表演纸牌魔术，聚光灯照亮，观众惊叹，魔术氛围，电影质感，4K，高质量
--neg blurry, low quality, distorted, extra limbs
```

**输出预期:**
- 时长：8秒
- 分辨率：1080p
- 宽高比：16:9
- 模型：Pika 1.0
- 运动强度：5

### 图生视频示例

**输入:**
- 参考图片：雪山风景照
- 提示词：`云层飘动，阳光洒在雪山上，壮丽自然，4K，高质量`
- 运动强度：6

**输出预期:**
- 时长：6秒
- 云层飘动效果
- 阳光变化
- 保持原图构图

### 视频扩展示例

**输入:**
- 原视频：5秒的人物走路视频
- 提示词：`人物继续走路，保持一致风格，4K，高质量`
- 扩展方向：向前/向后

**输出预期:**
- 总时长：10秒
- 无缝衔接
- 风格保持一致

### 一致角色示例

**输入:**
- 参考图片：角色肖像照
- 提示词：`这个角色在公园里散步，保持一致，4K，高质量`

**输出预期:**
- 时长：8秒
- 角色特征保持一致
- 自然的走路动作

## 常见问题

### Q: 如何保持角色一致？
A: 使用角色参考图，保持角色描述一致。

### Q: 视频时长可以延长吗？
A: 可以使用视频扩展功能，每次可扩展数秒。

### Q: 支持哪些宽高比？
A: 支持 16:9（横屏）、9:16（竖屏）、1:1（方形）。
