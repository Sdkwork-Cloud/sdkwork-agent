# 视频生成产品体系架构

## 概述

本体系对当前主流视频生成模型（可灵、Vidu、即梦、Sora、Google、Runway、Pika）的生成方式进行高度抽象，提取共同的产品化模式，构建高内聚低耦合的视频提示词生成器。

---

## 主流视频生成模型分析

### 1. 可灵 (Kling)

**核心能力：**

- 文生视频
- 图生视频
- 首尾帧控制
- 多帧分镜
- 运动控制

**产品特点：**

- 强运动控制
- 中文内容优化
- 稳定输出

### 2. Vidu

**核心能力：**

- 文生视频
- 图生视频
- 视频到视频
- 电影质感

**产品特点：**

- 高质量渲染
- 电影风格
- 细节丰富

### 3. 即梦 (Jimeng / Seedance 2.0)

**核心能力：**

- 文生视频
- 图生视频
- 剪映集成
- **首尾帧入口** - 首帧图 + prompt
- **全能参考入口** - 多模态组合输入
- **全能智能参考（多模态参考）** - 支持最多12种参考文件
- 一键剧本转短剧
- 小说秒变短片
- 动作捕捉与打斗优化
- 原生音画同步
- 自分镜与自运镜

**两种入口模式：**

| 入口类型         | 使用场景              | 触发条件                  |
| ---------------- | --------------------- | ------------------------- |
| **首尾帧入口**   | 只上传首帧图 + prompt | 只有首帧/尾帧 + 文本      |
| **全能参考入口** | 多模态组合输入        | 图+视频+音频+文本任意组合 |

**注意：** 智能多帧和主体参考在 Seedance 2.0 中无法选中。

**全能智能参考核心特性：**

- 支持文本、图片、视频、音频四种模态输入
- 最多可接入12份参考素材
- 精准复刻画面构图、角色特征、运镜逻辑、音乐氛围
- 角色零突变 - 跨镜头角色一致性保持
- 原生音画像素级对齐
- 嘴型精准同步、面部表情与音频节奏精准对齐
- 自动生成匹配的背景音乐与环境音效
- **可参考万物** - 动作、特效、形式、运镜、人物、场景、声音
- **强创意生成** + **指令响应精准**

**产品特点：**

- 稳定性强
- 剪映无缝集成
- 易用性好
- 生成速度提升超10倍
- 废片率大幅降低
- 可用率提升至90%以上
- 电影级视频生成
- 长视频流畅叙事

### 4. Sora

**核心能力：**

- 文生视频
- 长视频生成
- 一致性保持

**产品特点：**

- 长视频支持（可达60秒）
- 高度一致性
- 复杂场景理解

### 5. Google (Veo)

**核心能力：**

- 文生视频
- 角色一致性
- 首尾帧控制

**产品特点：**

- 角色一致性强
- Google 生态集成
- 多语言支持

### 6. Runway

**核心能力：**

- 文生视频
- 图生视频
- 视频到视频
- Motion Brush
- 首尾帧控制

**产品特点：**

- 专业工具集
- Motion Brush 局部运动控制
- Gen-3 最新模型

### 7. Pika

**核心能力：**

- 文生视频
- 图生视频
- 视频到视频
- 视频扩展

**产品特点：**

- 创意性强
- 易用性好
- Discord 和 Web 平台

## 高度抽象的产品体系

### 核心产品类型 (GenerationProduct)

| Product ID        | Product Name  | 描述                               | 核心能力                         |
| ----------------- | ------------- | ---------------------------------- | -------------------------------- |
| `VIDEO_TEXT`      | 文生视频      | 从文本描述生成视频                 | 纯文本输入 → 视频输出            |
| `VIDEO_IMAGE`     | 图生视频      | 从图片生成视频                     | 图片 + 文本 → 视频输出           |
| `VIDEO_FRAME`     | 首尾帧生成    | 从首尾帧生成过渡视频               | 首帧 + 末帧 → 过渡视频           |
| `VIDEO_MULTI`     | 多帧/分镜生成 | 从多个时间标签帧生成视频           | 多个分镜 → 完整视频              |
| `VIDEO_UNIVERSAL` | 全能智能参考  | 多模态参考生成（即梦Seedance 2.0） | 图片+视频+音频+文本 → 完整视频   |
| `VIDEO_REFERENCE` | 参考视频生成  | 基于参考视频生成                   | 参考视频 + 文本 → 新视频         |
| `VIDEO_STYLE`     | 风格迁移生成  | 基于风格参考生成视频               | 风格参考 + 内容 → 风格化视频     |
| `VIDEO_EXTEND`    | 视频扩展生成  | 扩展现有视频时长                   | 现有视频 + 方向 → 更长视频       |
| `VIDEO_EDIT`      | 视频编辑生成  | 编辑/修改现有视频                  | 现有视频 + 编辑指令 → 修改后视频 |

## 高内聚低耦合设计原则

### 1. 高内聚 (High Cohesion)

- 每个 GenerationProduct 负责单一类型的生成
- Asset Management 统一管理所有资源
- Platform Adaptation 集中处理平台差异

### 2. 低耦合 (Low Coupling)

- GenerationProduct 与 Platform 解耦
- Asset Management 独立于具体产品
- 各层之间通过标准化接口通信

## GenerationProduct 详细说明

### VIDEO_TEXT (文生视频)

**输入：** 纯文本描述
**输出：** 完整视频
**核心：** 场景描述 + 角色动作 + 相机移动

### VIDEO_IMAGE (图生视频)

**输入：** 参考图片 + 文本描述
**输出：** 基于图片的视频
**核心：** 保持图片一致性 + 添加自然运动

### VIDEO_FRAME (首尾帧生成)

**输入：** 首帧 + 末帧 + 过渡描述
**输出：** 从首帧到末帧的过渡视频
**核心：** 平滑插值 + 一致角色

### VIDEO_MULTI (多帧/分镜生成)

**输入：** 多个带时间标签的分镜
**输出：** 完整的分镜视频
**核心：** 时间线管理 + 分镜间过渡

### VIDEO_UNIVERSAL (全能智能参考 / 多模态参考)

**输入：** 文本 + 图片 + 视频 + 音频（最多12份参考素材）
**输出：** 完整的多模态参考视频
**核心：**

- 多模态融合：文本、图片、视频、音频四种模态输入
- 精准复刻：画面构图、角色特征、运镜逻辑、音乐氛围
- 角色一致性：跨镜头角色零突变
- 音画同步：原生音画像素级对齐、嘴型精准同步
- 自动音效：自动生成匹配的背景音乐与环境音效

**参考类型：**

- 角色图片：锁定人物长相、服装纹理、表情习惯
- 场景图片：复刻画面构图、环境氛围
- 参考视频：精准复刻运镜轨迹、动作细节
- 音频参考：匹配音乐节奏、口型同步
- 剧本/小说：一键剧本转短剧、小说秒变短片

**即梦Seedance 2.0 特性：**

- 最多支持12份参考素材
- 角色零突变：跨镜头一致性保持
- 原生音画同步：毫秒级匹配
- 自分镜与自运镜：自动规划镜头动作
- 生成速度提升超10倍
- 可用率提升至90%以上

### VIDEO_REFERENCE (参考视频生成)

**输入：** 参考视频 + 新描述
**输出：** 基于参考的新视频
**核心：** 参考提取 + 内容替换

### VIDEO_STYLE (风格迁移生成)

**输入：** 风格参考 + 内容描述
**输出：** 风格化视频
**核心：** 风格提取 + 风格应用

### VIDEO_EXTEND (视频扩展生成)

**输入：** 现有视频 + 扩展方向
**输出：** 更长的视频
**核心：** 前后无缝衔接

### VIDEO_EDIT (视频编辑生成)

**输入：** 现有视频 + 编辑指令
**输出：** 修改后的视频
**核心：** 局部修改 + 整体一致

## 扩展指南

### 添加新的 GenerationProduct

1. 在 product-architecture.md 中定义新的产品类型
2. 更新 SKILL.md 中的 product 枚举
3. 添加对应的生成逻辑和示例

### 添加新的平台

1. 创建 platform-[name].md 文件
2. 在 platform-adaptation 层添加适配逻辑
3. 更新平台列表

### 添加新的资源类型

1. 在 Asset Management 中定义新类型
2. 更新 assets 结构说明
3. 添加使用示例
