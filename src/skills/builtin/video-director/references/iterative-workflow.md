# 迭代式工作流程 (Iterative Workflow)

分步骤、迭代式的视频导演工作流程，逐个完成每个阶段，而不是一次性输出所有内容。

---

## 工作模式检测

### 自动检测模式

根据用户输入自动判断使用哪种工作模式：

| 输入特征 | 推荐模式 | 说明 |
|---------|---------|------|
| 用户明确说"一步步来"、"分步骤"、"迭代" | **迭代式** | 用户明确要求分步 |
| 用户输入剧本/故事，没有特殊要求 | **智能判断** | 根据内容复杂度选择 |
| 用户说"完整项目"、"一次性" | **完整模式** | 一次性输出所有内容 |
| 简短想法（< 100字） | **迭代式推荐** | 建议先做剧本 |
| 复杂项目（多场景、多角色） | **迭代式推荐** | 建议分步完成 |

### 迭代式关键词检测

| 关键词 | 说明 |
|--------|------|
| "一步步来"、"分步骤"、"迭代" | 明确要求迭代 |
| "先做剧本"、"先处理剧本" | 只做第1步 |
| "继续"、"下一步"、"然后呢" | 继续下一步 |
| "现在做角色"、"设计角色" | 跳到角色设计 |
| "做分镜"、"创分镜" | 跳转到分镜 |

## 迭代式工作流程（5个步骤）

### 步骤 0：确认工作模式

**输出：**
```
好的！我将使用迭代式工作流程来帮你完成这个视频项目。

我们将分5个步骤来完成：
1️⃣ 剧本处理与优化
2️⃣ 角色设计
3️⃣ 场景与道具设计
4️⃣ 分镜创作
5️⃣ 完整项目整合

是否现在开始第1步：剧本处理？
```

---

### 步骤 1：剧本处理（Step 1 - Script）

**输入：** 用户提供的任何内容（简短想法、故事、小说、剧本）

**输出内容：**
- 输入类型检测结果
- 剧本评估（如果适用）
- 完整的标准剧本
- 剧本优化建议

**输出格式示例：**
```json
{
  "step": 1,
  "stepName": "剧本处理",
  "status": "completed",
  "userInput": { ... },
  "script": { ... },
  "nextStep": {
    "step": 2,
    "stepName": "角色设计",
    "description": "基于剧本设计所有角色",
    "prompt": "是否继续进行第2步：角色设计？"
  }
}
```

**完成后询问：**
```
✅ 第1步完成！剧本已处理完毕。

是否继续进行第2步：角色设计？
```

---

### 步骤 2：角色设计（Step 2 - Characters）

**输入：** 步骤1的剧本

**输出内容：**
- 所有主要角色的完整设计
- 每个角色的参考图生成提示词

**输出格式示例：**
```json
{
  "step": 2,
  "stepName": "角色设计",
  "status": "completed",
  "script": { ... },
  "characters": [ ... ],
  "nextStep": {
    "step": 3,
    "stepName": "场景与道具设计",
    "description": "设计所有场景和道具",

}
```

**完成后询问：**
```
✅ 第2步完成！所有角色已设计完毕。

是否继续进行第3步：场景与道具设计？
```

---

### 步骤 3：场景与道具设计（Step 3 - Locations & Props）

**输入：** 步骤1的剧本 + 步骤2的角色

**输出内容：**
- 所有场景的完整设计
- 所有道具的完整设计

**输出格式示例：**
```json
{
  "step": 3,
  "stepName": "场景与道具设计",
  "status": "completed",
  "script": { ... },
  "characters": [ ... ],
```

```
✅ 第3步完成！所有场景和道具已设计完毕。

是否继续进行第4步：分镜创作？
```

**完成后询问：**
```
✅ 第4步完成！分镜已创作完毕。

是否继续进行第5步：完整项目整合？
```

---

### 步骤 5：完整项目整合（Step 5 - Complete Project）

**输入：** 步骤1-4的所有内容

**输出内容：**
- 完整的 FILM_PROJECT 结构
- 声音设计（可选）
- videoPromptEngineerInput

**输出格式示例：**
```json
{
  "step": 5,
  "stepName": "完整项目整合",
  "status": "completed",
  "project": {
    "id": "uuid-v4",
    "uuid": "uuid-v4",
    "type": "FILM_PROJECT",
    ...
  }
}
```

**完成后提示：**
```
🎉 恭喜！所有步骤已完成！

完整的电影导演项目已生成，你现在可以：
- 直接使用这个项目进行视频制作

- 根据需要修改任何部分
```

## 迭代控制功能

### 随时跳转到任意步骤

用户可以随时要求跳转到任意步骤：

| 用户请求 | 操作 |
|---------|------|
| "先做剧本" | 只执行步骤1 |
| "现在做角色" | 从步骤2开始 |
| "跳过场景，直接做分镜" | 跳到步骤4 |
| "重新做剧本" | 回到步骤1重新处理 |

### 随时修改任意部分

用户可以随时要求修改已完成的部分：

| 用户请求 | 操作 |
|---------|------|
| "修改角色小明" | 更新角色设计 |
| "调整场景1的灯光" | 更新场景设计 |
| "这个分镜不好，重新做" | 重新生成分镜 |

### 随时退出或暂停

| 用户请求 | 操作 |
|---------|------|
| "暂停"、"等一下" | 保存当前进度，等待下一步指示 |
| "先到这里" | 输出当前已完成的内容，暂停 |
| "下次再继续" | 保存进度，等待下次继续 |

## 最佳实践

### ✅ 推荐做法

1. **每步完成后明确询问**
   - 不要自动继续下一步
   - 给用户确认和修改的机会

2. **保持上下文连贯**
   - 每一步都基于前一步的输出
   - 确保 UUID 引用正确

3. **允许灵活跳转**
   - 用户可以随时跳到任意步骤
   - 用户可以随时修改任意部分

4. **提供清晰的进度指示**
   - 明确当前是第几步
   - 显示已完成和待完成的步骤

### ❌ 避免的做法

1. **不要一次性输出所有内容**
   - 除非用户明确要求"完整项目"

2. **不要忽略用户的跳转请求**
   - 用户说"先做剧本"就只做剧本
   - 不要自作主张继续下一步

3. **不要丢失上下文**
   - 每一步都要包含之前的所有数据
   - 确保引用完整性
