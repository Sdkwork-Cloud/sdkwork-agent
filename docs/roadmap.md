# 路线图

SDKWork Agent 的发展计划和未来方向。

## 当前版本 (v1.0)

### ✅ 已完成

- [x] DDD 分层架构
- [x] 微内核架构
- [x] OpenAI 兼容 API
- [x] Skill 系统（JS/TS/Python/Bash）
- [x] Tool 系统（分类、确认级别）
- [x] MCP 协议实现
- [x] Plugin 系统
- [x] 记忆系统（语义搜索）
- [x] 多 LLM Provider 支持
- [x] 完整文档和示例

## 短期计划 (v1.1 - v1.2)

### 🚧 进行中

- [ ] **Agent 市场**
  - Skill 市场
  - Tool 市场
  - Plugin 市场
  - 社区分享平台

- [ ] **可视化工具**
  - Agent 调试器
  - 执行链路可视化
  - 性能分析工具
  - 记忆浏览器

- [ ] **增强记忆**
  - 长期记忆存储（数据库支持）
  - 记忆压缩算法
  - 跨会话记忆共享
  - 记忆导入/导出

### 📋 计划内

- [ ] **更多 LLM 支持**
  - Azure OpenAI
  - AWS Bedrock
  - Cohere
  - AI21 Labs

- [ ] **高级 Skill 功能**
  - Skill 组合
  - Skill 模板
  - Skill 版本管理
  - Skill 热更新

- [ ] **安全增强**
  - Skill 沙箱
  - 权限系统
  - 审计日志
  - 敏感数据检测

## 中期计划 (v1.3 - v1.5)

### 🎯 目标

- [ ] **多 Agent 协作**
  - Agent 间通信
  - 任务分配
  - 协作工作流
  - 角色系统

- [ ] **知识库集成**
  - RAG 完整支持
  - 文档向量化
  - 知识图谱
  - 多数据源支持

- [ ] **工作流引擎**
  - 可视化工作流设计器
  - 条件分支
  - 循环执行
  - 错误处理

- [ ] **企业特性**
  - SSO 集成
  - 团队管理
  - 审计日志
  - SLA 监控

## 长期计划 (v2.0+)

### 🚀 愿景

- [ ] **自主 Agent**
  - 目标分解

  - 自我修正
  - 持续学习

- [ ] **多模态支持**
  - 图像理解
  - 语音交互

  - 文档解析

- [ ] **分布式架构**
  - Agent 集群

  - 故障转移

- [ ] **生态系统**
  - IDE 插件

  - 云服务

## 社区驱动

我们欢迎社区提出新功能和改进建议！

### 如何提议

1. 在 [GitHub Discussions](https://github.com/Sdkwork-Cloud/sdkwork-agent/discussions) 发起讨论
2. 描述你的使用场景
3. 解释为什么这个功能重要
4. 如果可能，提供实现思路

### 投票

- 👍 对你想要的功能投票
- 我们会优先实现票数高的功能
