## 报告安全漏洞

我们非常重视安全问题。如果你发现了安全漏洞，请通过以下方式报告：

### 报告方式

1. **私下报告**（推荐）
   - 发送邮件至: security@sdkwork.io
   - 主题格式: `[SECURITY] 漏洞描述`
   - 请提供详细的复现步骤和影响范围

2. **GitHub Security Advisories**
   - 使用 GitHub 的 [Security Advisories](https://github.com/sdkwork/agent/security/advisories) 功能

### 报告内容

请尽可能包含以下信息：

- 漏洞类型（例如：XSS、SQL注入、权限绕过等）
- 受影响的版本
- 详细的复现步骤
- 可能的修复建议
- 影响范围评估

### 响应时间

- **确认收到**: 24 小时内
- **初步评估**: 72 小时内
- **修复计划**: 1 周内
- **安全补丁发布**: 根据严重程度，2-4 周内

## 安全最佳实践

### 使用 SDKWork Agent 时的安全建议

1. **API 密钥管理**
   - 永远不要将 API 密钥硬编码在代码中
   - 使用环境变量或安全的密钥管理服务
   - 定期轮换 API 密钥

2. **Tool 确认级别**
   - 对于破坏性操作，使用 `confirm: 'destructive'`
   - 在生产环境中启用用户确认机制

3. **Skill 代码审查**
   - 审查所有自定义 Skill 的代码
   - 避免在 Skill 中执行敏感操作
   - 使用沙箱环境执行不可信代码

4. **输入验证**
   - 始终验证用户输入
   - 使用 Schema 验证 Skill 和 Tool 的输入

5. **日志和监控**
   - 启用详细的日志记录
   - 监控异常行为和错误率
   - 定期审计 Tool 调用记录
